# FixITPro v1.0.0-RC1 — Release Notes

**Release date:** 2026-06-01  
**Type:** Release Candidate  
**Platform:** SUNMI V2 Pro (Android) + Desktop Web (Next.js)  
**Stack:** NestJS 10 · Prisma 5 · PostgreSQL · Next.js 14 · Capacitor 5  

---

## What Is FixITPro?

A full-featured POS and repair-shop management system for mobile phone repair shops.  
Designed to run natively on the SUNMI V2 Pro handheld Android device and in any desktop browser.

## Phase 16 Additions (New in This RC)

### Smart Reminder & Anti-Forget System (Phase 16.1–16.7)

- **5 reminder types:** VIP_REPAIR, URGENT_REPAIR, PARTS_REQUEST_PENDING, TRANSFER_PENDING, PICKUP_WAITING
- **Server-side snooze:** 5 / 15 / 30 min per entity, with audit log
- **Per-user settings:** toggle each type on/off, OWNER scope (all branches vs. single)
- **Anti-spam gate:** 5-min sound cooldown per entity ID
- **5 Web Audio tone profiles** + CRITICAL pulsing overlay
- **CRITICAL reminders** cannot be dismissed without snoozing
- **New DB tables:** `ReminderSettings`, `ReminderSnooze` (migration `20260601000000`)

## Security Fixes (Phases 16.9–16.10)

| ID | Severity | Fix |
|----|----------|-----|
| C-1 | CRITICAL | Sales stock check moved inside transaction — atomic conditional `updateMany(WHERE quantity >= demand)` prevents concurrent oversell |
| C-2 | CRITICAL | Device history tenant isolation — non-SUPER_ADMIN callers with null tenantId now receive `403 Forbidden` instead of leaking cross-tenant data |
| M-1 | MAJOR | `POST /sales` now requires `@RequirePermission('sales.create')` — non-cashier roles blocked from direct API sale creation |
| M-4 | MAJOR | File upload validates MIME **and** extension whitelist; stored filename uses MIME-derived extension (prevents `shell.jpg.php` upload) |

---

## Data Integrity Fixes (Phases 16.9–16.11)

| ID | Severity | Fix |
|----|----------|-----|
| M-2 | MAJOR | Repair status transitions enforce explicit `ALLOWED` map — RECEIVED→COMPLETED (and all multi-step skips) now blocked |
| M-3 | MAJOR | Null product name in repair stock error replaced with `[ID: …]` fallback; required quantity added to message |
| N-3 | MINOR | Stock adjust OUT uses `updateMany(WHERE quantity >= demand)` — concurrent adjustments can no longer push stock negative |
| N-4 | MINOR | Warranty `warrantyDays < 1` blocked; `update` validates `endDate > startDate` |
| N-5 | MINOR | Debt partial payment preview uses integer-cent arithmetic — eliminates IEEE-754 float residuals in displayed balance |
